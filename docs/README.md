# Docs index

Project-level docs (`PROJECT_PLAN.md`, `RUNBOOK.md`, `FEATURE_REQUEST.md`,
`CLAUDE.md`, `README.md`) live in the repo root. This folder holds
supporting material that doesn't belong in those:

- **`reviews/`** — point-in-time independent reviews (architecture, security,
  UI/UX). Findings that turn into work get tracked in `FEATURE_REQUEST.md` or
  `PROJECT_PLAN.md`'s implementation log; these files stay as the original,
  dated source document.
- **`plans/active/`** — implementation plans for backlog work in progress,
  not yet folded into `FEATURE_REQUEST.md`'s Done log.
- **`plans/archive/`** — implementation plans for work that has since
  shipped, kept for historical reference.
